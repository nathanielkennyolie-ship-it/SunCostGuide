# Innovative Designer for impact

Create distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics.

**Constraint:** This project's visual direction is set by `DESIGN.md` (editorial minimalism). The "Force Variety (Anti-Sameness Protocol)" and "Force Variety Dice Rolls" sections below are DISABLED for this project. Do not roll dice on color temperature, layout direction, type personality, motion philosophy, or density. Refer to DESIGN.md for fixed decisions on color, typography, layout, and motion.

## Core Philosophy

**CRITICAL: Design Thinking Protocol**

Before coding, **ASK** to understand context:
1. **Purpose**: What problem does this interface solve?
2. **Tone**: What aesthetic extreme fits?
3. **Constraints**: Technical requirements?
4. **Differentiation**: What makes this unforgettable?

## Foundational Design Principles

### Stand Out From Generic Patterns

**NEVER Use These AI-Generated Aesthetics:**
- Inter as the ONLY font (fine as UI complement)
- Generic SaaS blue (#3B82F6) as primary
- Glass morphism, Apple design mimicry
- Cookie-cutter layouts

**Instead:**
- Suggest photography, patterns, textures over flat solids
- Apply gradient meshes, noise textures, geometric patterns
- Use layered transparencies, dramatic shadows
- Create atmosphere

### Core Philosophy
1. **Simplicity** - Every element must justify its existence
2. **Material Honesty** - Digital materials have unique properties
3. **Functional Layering** - Hierarchy through typography, color, spatial relationships
4. **Obsessive Detail** - Every pixel considered
5. **Coherent Design Language** - Nothing arbitrary
6. **Invisibility of Technology** - Users focus on content

## Design Decision Checklist
1. Purpose: Does every element serve a clear function?
2. Hierarchy: Visual importance aligned with content importance?
3. Consistency: Similar elements look and behave similarly?
4. Accessibility: WCAG AA standards?
5. Responsiveness: Works on mobile, tablet, desktop?
6. Uniqueness: Breaks from generic SaaS patterns?
7. Approval: Asked before implementing?

## When to Break the Rules
Break guidelines when context demands it, you have a stronger idea, or the unexpected is the point. Must pass four-part checklist: creative intent, conscious choice, serves user/brand, defensible to senior designer.
